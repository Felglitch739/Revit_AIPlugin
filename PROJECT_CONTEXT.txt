PROJECT NAME: Revit AIPlugin
DESCRIPTION: 
  Un plugin de Autodesk Revit que integra IA (Groq Chat API) para automatizar

3.2 Revit Add-in Backend (Revit_AIPlugin)
──────────────────────────────────────────
Location: Revit_AIPlugin/Revit/

Key Components:
  • RevitCommandDispatcher.cs
	- Implements IRevitCommandDispatcher
	- Bridges UI requests to Revit ExternalEvent handlers
	- Manages handler instances & polling
	- Methods queue commands and wait for results

  • Tools/ (Handler Directory)
	External Event Handlers (run on Revit's main thread):

	1. CrearMuroHandler.cs
	   - Creates single wall on specified level
	   - Inputs: nivel, longitud, altura, tipoMuro
	   - Finds wall type, creates line geometry, calls doc.Create.NewWall()
	   - Logs: entry, wall type lookup, line creation, success/error with duration

	2. LeerElementosHandler.cs
	   - Reads/lists elements by category
	   - Supports: Walls, Doors, Windows, Floors, Levels, Columns
	   - Outputs: Element name, level, ID (max 10 per result)
	   - Logs: entry, category validation, element count, details with duration

	3. CrearHabitacionEstructuradaHandler.cs
	   - Creates rectangular room perimeter
	   - Inputs: nivel, ancho, largo, altura
	   - Builds 4 walls from origin to width/length extents
	   - Logs: entry, level found, wall creation, duration

	4. ColocarMobiliarioHandler.cs
	   - Places furniture family instance
	   - Inputs: tipoMueble, x, y (meters)
	   - Searches OST_Furniture category
	   - Fallback: uses first available symbol if type not found
	   - Logs: entry, family search, point conversion, placement, duration

	5. ColocarPuertaHandler.cs
	   - Places host-based door on nearest wall
	   - Inputs: tipoPuerta, x, y (meters)
	   - Searches OST_Doors, finds nearest wall host
	   - Uses LocationCurve.Curve.Project() for proximity
	   - Logs: entry, family/host search, placement details, duration

	6. ColocarVentanaHandler.cs
	   - Places host-based window on nearest wall
	   - Inputs: tipoVentana, x, y (meters)
	   - Searches OST_Windows, finds nearest wall host
	   - Same wall-proximity logic as doors
	   - Logs: entry, family/host search, placement details, duration

4. LLM INTEGRATION (Groq Chat API)
================================================================================

API Details:
  - Endpoint: https://api.groq.com/openai/v1/chat/completions
  - Model: (specified in GroqService, typically mixtral or similar)
  - Auth: Bearer token (API key in secrets/config)

Prompt Engineering:
  - System prompt includes:
	* Role definition (AI architect assistant)
	* Available tools catalog
	* Spatial constraints (furniture layout rules)
	* Revit native family names
	* Coordinate system rules (Cartesian, strict spacing)
	* Collision avoidance guidelines
	* Perimeter placement rules for doors/windows

Tool Calling Flow:
  1. User inputs command (natural language) → MainWindow
  2. MainWindow sends system prompt + tools + user input → Groq
  3. Groq returns tool_calls (JSON) or text_response
  4. GroqService parses tool_calls → RevitToolExecutor
  5. RevitToolExecutor maps to dispatcher methods
  6. Dispatcher queues external events → Revit handlers execute
  7. Handlers log details + return result strings
  8. Results shown in UI

Units:
  - User-facing: METERS
  - Revit internal: Decimal feet (converted via UnitUtils)
  - Handlers auto-convert meter inputs to internal units

5. ARCHITECTURAL PATTERNS
================================================================================

5.1 External Event Pattern (Revit Safety)
──────────────────────────────────────────
Why: Revit API is not thread-safe; all operations must run on main thread.

Implementation:
  - Each handler implements IExternalEventHandler
  - RevitCommandDispatcher creates ExternalEvent wrappers
  - UI sets handler properties → raises event → polls for result
  - EsperarResultado() helper waits with timeout

5.2 Singleton Logger
────────────────────
  - RevitAILogger uses double-checked locking
  - Initialize() called once (idempotent)
  - All handlers call RevitAILogger.Info/Warn/Error/Debug
  - Close() called on app shutdown

5.3 Tool Definition → Executor → Dispatcher → Handler Chain
────────────────────────────────────────────────────────────
  ToolDefinitions (schema) 
	→ GroqService (LLM exposure)
	→ RevitToolExecutor (routing + parsing)
	→ IRevitCommandDispatcher (interface)
	→ RevitCommandDispatcher (queue + event)
	→ IExternalEventHandler (Revit execution)

6. KEY TECHNICAL DECISIONS
================================================================================

1. Serilog for Logging
   - Structured, multi-sink, easy filtering
   - Production-ready + performant

2. Groq API Choice
   - Fast inference, multi-model support
   - Tool calling capability (native JSON schema support)

3. ExternalEvent-based Dispatch
   - Only safe way to call Revit API from UI thread
   - Avoids crashes/freezes in Revit

4. Family Fallback Strategy
   - If requested type not found → use first available symbol
   - Prevents command failure on missing family
   - Logged as Warn for visibility

5. Nearest-Wall Host Finding
   - Uses LocationCurve.Curve.Project(point) for proximity
   - Avoids invalid/complex Revit geometry APIs
   - Simple, reliable, works across versions

6. Metric System (Meters)
   - UI/AI uses meters for clarity
   - Automatic conversion to Revit internal units
   - Consistent across all placement commands

7. DATA FLOW EXAMPLE (User Command → Revit)
================================================================================

User: "Crea una puerta de vidrio en las coordenadas 5,3"
	   (Create a glass door at coordinates 5,3)

Flow:
  1. MainWindow receives input
  2. Builds system prompt (spatial rules, available tools)
  3. Sends to Groq: 
	 {
	   "system": [...constraint rules...],
	   "messages": [{"role": "user", "content": "..."}],
	   "tools": [{"name": "ColocarPuerta", "parameters": {...}}, ...]
	 }
  4. Groq responds:
	 {
	   "choices": [{
		 "message": {
		   "tool_calls": [{
			 "function": {
			   "name": "ColocarPuerta",
			   "arguments": "{\"tipoPuerta\":\"Glass Door\",\"x\":5.0,\"y\":3.0}"
			 }
		   }]
		 }
	   }]
	 }
  5. GroqService parses → RevitToolExecutor extracts args
  6. RevitToolExecutor.ExecuteColocarPuertaAsync("Glass Door", 5.0, 3.0)
  7. Dispatcher.ColocarPuerta("Glass Door", 5.0, 3.0)
  8. Creates ColocarPuertaHandler, sets properties, queues ExternalEvent
  9. Handler runs on Revit main thread:
	 - Converts 5.0m, 3.0m to internal units (feet)
	 - Searches OST_Doors for "Glass Door" family
	 - Finds nearest wall to (5, 3) point
	 - Calls doc.Create.NewFamilyInstance(point, symbol, wall, NonStructural)
	 - Logs: "✅ Puerta colocada: Glass Door en X=5m, Y=3m (Duracion: 142ms)"
  10. Result string returned to UI, displayed to user

Logs generated during this operation:
  - revit-ai-2025-01-15.log: [Info] Iniciando ColocarPuerta...
  - revit-ai-2025-01-15.log: [Debug] Familia encontrada: Doors:Glass Door
  - revit-ai-2025-01-15.log: [Info] ✅ Puerta colocada...

12. HOW TO EXTEND
================================================================================

Adding a New Revit Command (e.g., ColocarViga/Place Beam):

1. Add Tool Definition (ToolDefinitions.cs):
   public static ToolDefinition ColocarViga => new(
	 "ColocarViga",
	 "Coloca una viga estructural",
	 new { tipoViga = "string", x = "double", y = "double", ... }
   );

2. Add Dispatcher Method (IRevitCommandDispatcher.cs):
   Task<string> ColocarViga(string tipoViga, double x, double y);

3. Implement in Dispatcher (RevitCommandDispatcher.cs):
   public Task<string> ColocarViga(string tipoViga, double x, double y) { ... }

4. Create Handler (Tools/ColocarVigaHandler.cs):
   public class ColocarVigaHandler : IExternalEventHandler { ... }

5. Add Executor Route (RevitToolExecutor.cs):
   case "ColocarViga":
	 return await ExecuteColocarVigaAsync(args);

6. Update Prompt (PromptTemplates.cs):
   Add ColocarViga to tool hints and spatial constraints if needed.

7. Test & log entries will automatically be generated via RevitAILogger.
